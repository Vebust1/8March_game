# Угадай мелодию

Игра «Угадай мелодию» — веб-приложение для проведения викторины с угадыванием мелодий в реальном времени. Ведущий управляет игрой, игроки отвечают на вопросы и нажимают кнопку «Ответить», зрители следят за счётом на большом экране.

## Быстрый старт

### Требования

- **Node.js** 18+ (рекомендуется LTS)

### Установка и запуск

```bash
# Установка зависимостей
npm install

# Запуск сервера
npm start
```

По умолчанию приложение доступно по адресу: **http://localhost:3001**

Для разработки с авто-перезапуском при изменениях:

```bash
npm run dev
```

### Переменные окружения

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `PORT`     | Порт сервера | `3001` |

## Экраны и роли

Приложение использует единый `index.html`, а интерфейс выбирается по URL.

| URL | Роль | Описание |
|-----|------|----------|
| `/valdis` | **Ведущий** | Управление игрой: старт, выбор вопросов, оценка ответов, следующий раунд |
| `/player` | **Игрок** | Регистрация, выбор вопросов (когда на очереди), кнопка «Ответить», пробел для быстрого ответа |
| `/` | **Зритель** | Просмотр игрового поля и счёта в режиме только чтение |

### Как подключаться

- **Ведущий**: открывает `/valdis` и запускает игру.
- **Игроки**: открывают `/player` с телефона/планшета, вводят имя и нажимают «Войти в игру».
- **Зрители**: открывают `/guest` на проекторе или большом экране.

Все участники должны находиться в одной сети и использовать один и тот же адрес сервера (например, `http://192.168.1.100:3001`).

## Раунды и типы игры

Раунды задаются в `questions/rounds.config.json`:

| Тип раунда | Описание | Параметры |
|------------|----------|-----------|
| `fixed` | Обычные вопросы с фиксированным временем | `timePerQuestion` — секунд на ответ (по умолчанию 30) |
| `countdown` | Чем быстрее ответ — тем больше очков | `totalTime` — секунд на таймере, `multiplier` — множитель очков |
| `auction` | Аукцион: ведущий выбирает, кто ответил правильно | `pointsPerQuestion` — очки за вопрос |
| `blitz` | Блиц: серия мелодий, побеждает тот, кто угадал больше всего | `pointsForWinner` — очки победителю |

Пример `rounds.config.json`:

```json
{
  "rounds": [
    {
      "id": "round-1",
      "type": "fixed",
      "folder": "round-1",
      "params": { "timePerQuestion": 30 }
    },
    {
      "id": "round-2",
      "type": "countdown",
      "folder": "round-2",
      "params": { "totalTime": 30, "multiplier": 1 }
    },
    {
      "id": "round-3",
      "type": "auction",
      "folder": "round-3",
      "params": { "pointsPerQuestion": 20 }
    },
    {
      "id": "round-4",
      "type": "blitz",
      "folder": "blitz-test",
      "params": { "pointsForWinner": 50 }
    }
  ]
}
```

## Как составлять вопросы

### Обычные раунды (fixed, countdown, auction)

Вопросы лежат в папке `questions/<folder>/`, где `folder` — значение из `rounds.config.json`.

**Структура папок:**

```
questions/
└── round-1/                    ← folder из конфига
    ├── meta.json               ← опционально, общие настройки раунда
    ├── lyubov/                 ← категория (id)
    │   ├── meta.json           ← название категории
    │   ├── q-10/               ← вопрос на 10 очков
    │   │   └── meta.json
    │   ├── q-20/
    │   │   └── meta.json
    │   └── q-30/
    │       └── meta.json
    └── goroda-i-strany/
        ├── meta.json
        └── q-10/
            └── meta.json
```

**Категория** — папка с `meta.json`:

```json
{
  "id": "lyubov",
  "title": "Любовь",
  "order": 2
}
```

- `id` — идентификатор (обычно совпадает с именем папки)
- `title` — отображаемое название
- `order` — порядок категорий (по возрастанию)

**Вопрос** — папка `q-<очки>` (например, `q-10`, `q-20`, `q-30`) с файлом `meta.json`:

```json
{
  "id": "lyubov-q-30",
  "points": 30,
  "questionBlocks": [
    { "type": "text", "content": "«Кто поёт?»" },
    { "type": "audio", "file": "intro.mp3" }
  ],
  "answerBlocks": [
    { "type": "text", "content": "Дмитрий Хворостовский" }
  ]
}
```

| Поле | Обязательно | Описание |
|------|-------------|----------|
| `id` | да | Уникальный идентификатор вопроса (например, `lyubov-q-30`) |
| `points` | да | Количество очков за вопрос |
| `questionBlocks` | да | Массив блоков вопроса |
| `answerBlocks` | нет | Правильный ответ (для ведущего) |

**Типы блоков в questionBlocks и answerBlocks:**

| type | Поля | Описание |
|------|------|----------|
| `text` | `content` | Текст вопроса или ответа |
| `audio` | `file` | Файл мелодии (mp3, wav и т.п.) в папке вопроса |
| `image` | `file` | Изображение (png, jpg и т.п.) в папке вопроса |

Файлы аудио и изображений кладутся в папку вопроса рядом с `meta.json`. Например, `questions/round-1/lyubov/q-30/intro.mp3`, `questions/round-1/lyubov/q-30/hint.png`.

**Пример вопроса с изображением:**

```json
{
  "id": "img-q-20",
  "points": 20,
  "questionBlocks": [
    { "type": "text", "content": "Угадайте по картинке" },
    { "type": "image", "file": "hint.png" }
  ],
  "answerBlocks": [
    { "type": "text", "content": "Правильный ответ" },
    { "type": "image", "file": "answer.png" }
  ]
}
```

**Минимальный пример вопроса** (только текст):

```json
{
  "id": "rock-q-10",
  "points": 10,
  "questionBlocks": [
    { "type": "text", "content": "«Группа крови на рукаве...»" }
  ],
  "answerBlocks": [
    { "type": "text", "content": "Группа крови — Кино" }
  ]
}
```

### Раунд-блиц (blitz)

Блиц хранится в `questions-blitz/<folder>/`. Структура:

```
questions-blitz/
└── blitz-test/
    ├── meta.json
    └── tracks/           ← папка с аудиофайлами
        ├── 01.mp3
        ├── 02.mp3
        └── ...
```

**meta.json блица:**

```json
{
  "id": "blitz-test",
  "trackCount": 8,
  "pointsForWinner": 50,
  "tracks": [
    { "file": "01.mp3", "answer": "Чёрный бумер — Серега" },
    { "file": "02.mp3", "answer": "На заре — Альянс" },
    { "file": "03.mp3", "answer": "Владимирский централ — Михаил Круг" }
  ]
}
```

| Поле | Описание |
|------|----------|
| `id` | Идентификатор блица |
| `trackCount` | Количество треков (можно не указывать, если заполнен `tracks`) |
| `pointsForWinner` | Очки победителю |
| `tracks` | Массив объектов `{ "file": "имя.mp3", "answer": "Ответ" }` |

Файлы из `tracks` должны лежать в `questions-blitz/<folder>/tracks/`.

## Использование во время игры

### Ведущий (`/valdis`)

1. Нажать **«Начать игру»** — старт первого раунда.
2. **Выбор вопроса**: клик по ячейке на игровом поле или кнопка ★ (случайный вопрос с минимальными очками).
3. Когда игрок нажал «Ответить»: выбрать **«Правильно»** или **«Неправильно»**.
4. В раунде **auction**: выбрать игрока, который ответил правильно, или **«Неправильно»**.
5. В **блице**: воспроизвести треки вручную (или через своё ПО), в конце выбрать победителя или **«След. раунд»**.
6. **«След. раунд»** — переход к следующему раунду или завершение игры.

### Игрок (`/player`)

1. Ввести имя и нажать **«Войти в игру»**.
2. Когда очередь выбрать вопрос — кликнуть по ячейке или кнопке ★.
3. После начала вопроса нажать **«Ответить»** (или пробел), чтобы ответить первым.
4. В раундах `fixed` и `countdown` кнопка «Ответить» доступна при активном вопросе; в `auction` вопрос выбирает только ведущий.

### Зритель (`/guest`)

Только просмотр игрового поля и счёта в реальном времени. Взаимодействия нет.

## API и WebSocket

### REST API

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/config` | Конфигурация раундов |
| GET | `/api/round/:roundId` | Данные раунда (категории, вопросы) |

### WebSocket (Socket.io)

События, генерируемые сервером:

- `game:state` — актуальное состояние игры
- `game:round-started` — старт раунда
- `game:question-started` — старт вопроса (в т.ч. `audioUrl` для воспроизведения)
- `game:countdown` — оставшееся время (раунд countdown)
- `game:question-ended` — конец вопроса
- `game:buzz` — игрок нажал «Ответить»
- `game:play-audio` — команда воспроизведения аудио (от ведущего)
- `game:ended` — конец игры
